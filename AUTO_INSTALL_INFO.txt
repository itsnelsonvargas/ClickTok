How does auto-installation work?

1. ClickTok.bat checks: python -c "import moviepy"
2. If import fails â†’ Dependencies missing
3. Calls: setup.bat automatically
4. setup.bat runs: python setup.py
5. setup.py installs all packages
6. Returns to ClickTok.bat
7. ClickTok.bat launches: python main.py
8. main.py double-checks dependencies
9. If still missing â†’ Shows error with instructions
10. If installed â†’ Launches GUI!

Additionally, main.py also has auto-install:
- If you run: python main.py directly
- It also checks and auto-installs if needed
- Works from both .bat file AND command line

Key changes to enable auto-install:

1. Moved imports inside functions
   - Before: Imports at module level
   - After: Imports inside launch_gui() and launch_cli()
   - Why: Allows checking deps before importing

2. Early dependency check
   - check_dependencies() runs before heavy imports
   - Tests for critical packages (moviepy, playwright, etc.)
   - Runs setup.py automatically if missing

3. Subprocess installation
   - subprocess.run([sys.executable, "setup.py"])
   - Waits for completion
   - New Python subprocess sees newly installed packages

4. Post-install imports
   - After installation completes
   - Imports happen for first time in current process
   - Works because imports are deferred to functions
